# AI Blog Generator - Structured Pipeline Instructions

## Mode: Structured Blog Generation

This document defines the strict 5-step structured blog generation pipeline.

### Global Rules (STRICT)
- Output must be ONLY plain text (NO JSON)
- Each step MUST start with: `STEP: <Step Name>`
- Do NOT skip any step
- Do NOT merge steps
- Do NOT jump ahead
- Follow formats EXACTLY
- Planner MUST output in table format
- Generator MUST strictly follow Planner structure
- Editor MUST only refine (NOT restructure)

---

## STEP 1: ROUTER

**Purpose:** Classify the request and extract core blog parameters.

**Output Format:**
```
STEP: Router

Task: blog
Topic: <extracted from user input>
Tone: <informative / casual / professional / technical>
Depth: <short / medium / long>
```

**Rules:**
- Map user tone to: informative, casual, professional, or technical
- Map complexity to: short (800 words), medium (1200 words), long (1800+ words)
- Extract topic exactly as user provided

---

## STEP 2: PLANNER

**Purpose:** Create a structured blog plan with sections and key points.

**Output Format:**
```
STEP: Planner

| Section No | Section Name     | Description                          | Key Points Included              |
|------------|----------------|--------------------------------------|----------------------------------|
| 1          | Introduction   | Overview of the topic                | Definition, context              |
| 2          | Core Concept   | Explanation of main idea             | Key mechanisms, explanation      |
| ...        | ...            | ...                                  | ...                              |
```

**Rules:**
- Output ONLY the markdown table (no extra text before/after)
- Adjust section names based on topic (don't use generic ones)
- Keep total sections between 5–8
- Include Introduction and Conclusion
- Avoid redundant sections
- Each row should have clear, concise descriptions

---

## STEP 3: RESEARCH

**Purpose:** Generate useful supporting information for the blog.

**Output Format:**
```
STEP: Research

Key Facts:
- Fact 1
- Fact 2
- Fact 3

Statistics:
- Statistic 1
- Statistic 2

Insights:
- Insight 1
- Insight 2

Search Queries Used:
- Query 1
- Query 2
```

**Rules:**
- Provide 3-5 items per category
- Use Tavily API to fetch real research data
- Format each item as a bullet point
- Include sources/references where available
- If no data available, provide domain knowledge

---

## STEP 4: GENERATOR

**Purpose:** Write the full blog strictly based on Planner structure.

**Output Format:**
```
STEP: Generator

# <Blog Title>

## Section 1 Name
Content here...

## Section 2 Name
Content here...

...

## Conclusion
Content here...
```

**Rules:**
- Follow section order EXACTLY from Planner
- Do NOT add/remove sections
- Do NOT merge sections
- Use Research data where relevant
- Maintain consistent tone throughout
- Each section must have 150-300 words
- Use natural language and examples
- Cover all bullet points from Planner

---

## STEP 5: EDITOR

**Purpose:** Refine and improve the blog output.

**Output Format:**
```
STEP: Editor

# <Final Blog Title>

## Section 1 Name
Final polished content...

## Section 2 Name
Final polished content...

...

## Conclusion
Final polished content...
```

**Rules:**
- Improve clarity, flow, readability, and SEO
- Do NOT change structure
- Do NOT remove sections or subsections
- Do NOT reorder sections
- Fix grammar, spelling, and punctuation
- Enhance examples and explanations
- Ensure consistent formatting
- Add smooth transitions between sections
- Final output should be publication-ready

---

## Implementation Details

### Node Execution Order
1. Router Node → Classify topic, determine tone and depth
2. Planner Node → Create section structure in table format
3. Research Node → Gather facts and statistics
4. Generator Node → Write full blog following plan
5. Editor Node → Polish and refine final output

### Configuration
- Mode: "structured" (vs. default "auto")
- Output Type: "plaintext" (not JSON)
- Format Strict: True

### Initialization
```
initial_state = {
    "topic": "<user input>",
    "mode": "structured",
    "router_output": "",
    "planner_output": "",
    "research_output": "",
    "generator_output": "",
    "final_output": "",
}
```

---

## Notes
- This is a strict sequential pipeline - no parallel processing
- Each step builds on previous outputs
- Output validation checks for required format markers
- Plain text only - no JSON, markdown formatting, or metadata
